Holly fleet · manual tools

When you need to touch one PC directly

The normal path is AWS: code arrives on its own, ./greengrass/publish.sh updates everything. These two tools are the manual fallbacks — for when AWS is down, a PC needs code by hand, or an old PC never got SSH. Back to the menu.

push.sh — push code to one PC (or all) over Tailscale

When to use AWS/Greengrass is unreachable, or you want to get code onto a single PC right now without publishing a release — e.g. quick testing on one machine.

Run it on your Mac, from the pc-setup folder. It copies the holly app code straight to the PC over the tailnet using SSH:

Mac terminal
./tools/push.sh list         # first: see which PCs are reachable right now
./tools/push.sh holly-002     # push to just that PC
./tools/push.sh              # push to every PC in the list

Three things it needs:

  • The PC is online in Tailscale (check the admin console)
  • The PC has SSH set up — every PC provisioned by setup.ps1 has it; older PCs may need the retrofit below
  • The PC's name is in the NUCS list at the top of push.sh — add new PCs there once
After a push, the running app is still on the old code. Close it once (over VNC) — the watchdog relaunches it on the new code within seconds. Or wait for tomorrow's 08:30 launch.

aws_onboard.ps1 — put an older PC onto AWS

When to use A PC that was fully set up before AWS was part of setup.ps1 (PCs 001–003). One run catches it up: registers the watchdog task and enrols it into Greengrass so code arrives automatically. New PCs never need this — setup.ps1 does it all.

⬇  Download aws_onboard.ps1

On the PC (get the file there via this page or VNC): right-click Start → Terminal (Admin):

Terminal (Admin) on the PC
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
cd C:\Users\holly\Downloads
.\aws_onboard.ps1 -PCNumber 001 -AwsAccessKey "AKIA..." -AwsSecretKey "..."

Afterwards the PC shows Healthy in AWS (IoT Core → Greengrass → Core devices), pulls the current code into C:\code\holly by itself, and the watchdog starts at the next logon — reboot when convenient.

ssh_script.ps1 — give one PC SSH access

When to use push.sh can't reach a PC because it has no SSH — a machine set up before SSH was part of setup.ps1, or one where Windows was reinstalled. You only ever run this once per PC. New PCs never need it.

Get the script onto the PC (download it there from this site, or copy it over VNC):

⬇  Download ssh_script.ps1

Then on that PC: right-click Start → Terminal (Admin), and run:

Terminal (Admin) on the PC
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
cd C:\Users\holly\Downloads
.\ssh_script.ps1

It installs the built-in Windows SSH server, sets it to start on boot, and authorises your Mac's key. Takes under a minute. Then test from your Mac:

Mac terminal
ssh -i ~/.ssh/fleet_deploy holly@holly-002

If you get a PowerShell prompt on the PC, it worked — push.sh can now reach that machine.
