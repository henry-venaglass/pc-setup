Setting up a new Holly PC

One script does everything. You type one command, wait about 20 minutes, and a fresh NUC becomes a fully working Holly kiosk PC. This page explains what it does and exactly how to run it.

1What the script actually does

You run setup.ps1 once on a brand-new PC. In plain terms it does five jobs:

🧹

Cleans up Windows

Removes the junk apps, ads, OneDrive, Cortana — everything that gets in the way of a kiosk.

🔒

Locks it down

Names the PC HOLLY-xxx, makes it log in by itself, turns off wifi and Bluetooth, stops Windows interrupting.

🛠️

Installs the tools

Tailscale (remote network), VNC (see the screen remotely), SSH (remote terminal), OBS, git, and uv (runs the app).

🐕

Sets up the watchdog

A little guardian that opens the app at 08:30, closes it at 18:00 (weekdays only), and restarts it in seconds if it ever crashes or freezes.

☁️

Connects it to AWS

The PC joins the fleet and pulls the app code down by itself — you never copy code onto it. Later, one command on the Mac (./publish.sh) updates every PC in the fleet at once.

2What you need before you start

  • The NUC with power, monitor, keyboard, mouse, and an ethernet cable (never wifi)
  • setup.ps1 — download it straight from this page in step 4 (or bring it on a USB stick)
  • This PC's number — e.g. 004
  • The Tailscale key (starts tskey-auth-…) — from the Tailscale admin console
  • The AWS key pair (starts AKIA… plus its secret) — same one for every PC, used only during setup and wiped from the PC afterwards
  • A label maker

3First: get through Windows setup, offline

Power on. When Windows asks about the internet, click "I don't have internet" — do not connect to anything yet. If Windows refuses to continue, press Shift+F10, type OOBE\BYPASSNRO, press Enter, and carry on offline.

  • Username: holly (all lowercase) · Password: holly
  • Security questions: answer holly to all three
  • Say No to every privacy question

Once you reach the desktop, plug in the ethernet cable.

4Get the script, then run it

On the new PC, open this page in the browser and click:

⬇  Download setup.ps1

(If it opens as text instead of downloading, right-click the button → Save link as… Make sure it ends up in the Downloads folder. USB stick works too.)

Then right-click the Start button → Terminal (Admin), and run these three lines, one at a time:

Terminal (Admin)
Set-ExecutionPolicy -Scope Process -ExecutionPolicy Bypass
cd C:\Users\holly\Downloads
.\setup.ps1 -PCNumber 004 -AuthKey "tskey-auth-..." -AwsAccessKey "AKIA..." -AwsSecretKey "..."

The four highlighted bits are the only things you ever change:

-PCNumberThis PC's number. The only value that's different per PC.
-AuthKeyThe Tailscale key. Same for every PC (until it expires).
-AwsAccessKeyThe AWS key pair. Same for every PC. Only used during setup, then wiped from the machine.
-AwsSecretKey

It takes 15–30 minutes and prints what it's doing as it goes. At the end it shows a summary — you want READY (a few warnings are usually fine; failures are not). Say yes when it asks to reboot.

After the reboot, open Settings → Windows Update and keep installing updates + rebooting until there's nothing left.

5Check it worked

From your Mac (not the NUC):

  1. Tailscale admin console — holly-004 is listed and online
  2. AWS console → IoT Core → Greengrass → Core devices — holly-004 says Healthy
  3. Wait a few minutes — the app code appears on the PC at C:\code\holly by itself

Then two small manual jobs on the NUC:

  1. Open OBS once → choose "I will only be using the virtual camera" → accept defaults
  2. Open Device Manager → disable the built-in microphones, speakers, and webcam

Finally, restart the PC and just watch. It should log itself in with no password, and — if it's a weekday between 08:30 and 18:00 — the app should appear on its own within a minute or so. Stick a HOLLY-004 label on the bottom of the box. Done.

That's the whole job. No copying code, no configuring schedules, no setting up the app — the script and AWS handle all of it.

6Everyday stuff, after setup

To update the app on every PC: on the Mac, run ./publish.sh. All PCs download the new version themselves — even ones that are off catch up when they come back. The running app switches to the new code the next time it starts (tomorrow 08:30, or close it once and the watchdog reopens it on the new version).

To look at a PC remotely: TightVNC Viewer to its tailnet address (see the screen), or SSH (terminal).

If the app crashes mid-day: nothing to do — the watchdog restarts it within seconds.

If something goes wrong during setup: the script prints a summary of exactly what failed, and saves a log on the Desktop. Fix the issue (usually network) and just run the same command again — it's safe to re-run.
